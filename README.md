# @NetCourseWork_bot — Telegram-бот для изучения английских слов

Это курсовая работа «ТГ-чат-бот «Обучалка английскому языку»» по курсу «Базы данных»
Хранит данные в PostgreSQL с использованием SQLAlchemy ORM.

## Описание
Бот помогает учить английские слова:  
- Добавить карточку (/add)  
- Cлучайная карточка (/next)  
- Удалить карточку (/delete)
- Статистика (/stats)  
- Редактирование карточки (кнопка «Изменить»)  
- Меню и inline-кнопки

## Технологии
- Python 3  
- pyTelegramBotAPI  
- SQLAlchemy + psycopg2-binary  
- PostgreSQL  
- python-dotenv

## Cтруктура проекта
- main.py — основная логика бота, обработка команд и сообщений.
- database.py — конфигурация подключения к базе данных и настройки сессий.
- models.py — описание структуры таблицы WordCard (модели базы данных).
- .env — файл для хранения чувствительных данных (токен, параметры БД).
- .gitignore — исключение системных файлов и секретов из репозитория.

## Установка и настройка
1. Клонируйте репозиторий:
git clone https://github.com/DenisKorablOff/tg_bot_coursework.git
cd tg_bot_coursework

2. Создайте виртуальное окружение (рекомендуется):
python -m venv .venv
source venv/bin/activate  # Для Linux/macOS
venv\Scripts\activate     # Для Windows

3. Cкачивание и установка библиотек (инструментов) необходимых для работы:
pip install pyTelegramBotAPI python-dotenv sqlalchemy psycopg2-binary

4. Создайте файл .env:
* В корне проекта создайте файл с именем .env (точка в начале важна).
* Этот файл содержит конфигурационные переменные (секреты, такие как токен бота и пароль от базы данных).
* Содержимое .env (напишите вручную, не копируйте — чтобы избежать скрытых символов):
  - TOKEN=твой_токен_бота_от_BotFather
  - DB_USER=твой_логин_в_PostgreSql
  - DB_PASSWORD=твой_пароль_в_PostgreSql
  - DB_HOST=хост_в_PostgreSql
  - DB_PORT=порт_в_PostgreSql
  - DB_NAME=название_базы_данных_в_PostgreSql
* Важно:
Замените значения на свои (токен — от BotFather, пароль — ваш реальный от PostgreSQL).
Если пароль содержит спецсимволы (апострофы, кавычки) — наберите его вручную, чтобы избежать проблем с кодировкой (UnicodeDecodeError).
Не добавляйте пробелы вокруг = и не используйте кавычки вокруг значений, если в них нет пробелов.

5. Создайте базу данных в PostgreSQL:
- Используйте DBeaver или pgAdmin.
- Создайте новую базу с именем english_words_bot (или тем, что указали в .env).
- Убедитесь, что пользователь postgres (или указанный в .env) имеет доступ к базе с паролем из .env.

## Запуск
1. Запустите скрипт:
* python main.py
2. В консоли увидите Bot is running — бот готов.
3. В Telegram найдите бота по username (например, @NetCourseWork_bot) и напишите /start.

## Использование программы
1. /start — Открывает меню с кнопками для удобного навигации.
2. /add (или кнопка "Добавить слово") — Пошаговое добавление новой карточки: английское слово → русский перевод → транскрипция (опционально) → пример предложения (опционально).
- Бот спрашивает каждое поле по очереди.
- После добавления показывает ID карточки.
5. /next (или кнопка "Показать случайное") — Показывает случайную карточку из базы с кнопками: "Следующая карточка", "Повторить эту", "Удалить эту", "Изменить".
- Если база пуста — предложит добавить слово.
7. /delete — Удаление карточки по ID (бот спросит ID).
8. /stats (или кнопка "Статистика") — Показывает общее количество карточек в базе.
9. /help (или кнопка "Помощь") — Список всех команд и функций.

## Пример работы

1. /add → Бот: "Напиши слово на английском:"
- Вы: "hello"
- Бот: "Теперь перевод на русский:"
- Вы: "привет"
- ... (и так дальше).
2. /next → Бот показывает карточку с текстом и кнопками.
3. Нажатие "Изменить" — Бот начинает редактирование: "Новое английское слово (или Enter):"

## Возможные ошибки и отладка

* Если бот не запускается: Проверьте .env на правильность (без пробелов вокруг =, без кавычек в значениях, без скрытых символов).
* Если ошибка подключения к базе: Убедитесь, что база существует, пароль верный, PostgreSQL запущен.
* Если сессия ( /add или /edit ) "зависла" — напишите /start или другую команду — бот завершит сессию.
